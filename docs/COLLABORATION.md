# Collaboration Framework

## Our Roles & Responsibilities

### Your Role (Developer)
- **Execute all code changes**: You manually write, edit, find-and-replace code
- **Manual testing**: Run commands, test features, verify outputs
- **Decision making**: Final call on design choices and implementation direction
- **Understanding**: Take time to understand each step—don't just copy-paste blindly
- **Feedback**: Report what works, what breaks, what's unclear

### My Role (Guide & Architect)
- **Provide step-by-step instructions**: Break down tasks into small, actionable steps
- **Supply exact code snippets**: Give you exact file paths, line numbers, code to copy/paste
- **Explain the WHY**: Clarify what each step does and why it matters
- **Flag issues**: Identify problems, suggest fixes, but don't implement them
- **Document progress**: Maintain this document so we always know where we are
- **Suggest alternatives**: When choices exist, present tradeoffs clearly

## Workflow Structure

Each phase follows this pattern:

1. **Inspect**: Understand current state
2. **Plan**: Break into concrete steps with code
3. **You Execute**: Make the changes manually
4. **Verify**: Confirm it works
5. **Document**: Update progress tracker
6. **Move Forward**: Next phase

## Communication Style

- **My instructions**: Step-by-step numbered lists with file paths and line numbers
- **Your feedback**: What succeeded, what failed, what was unclear
- **Progress tracking**: Real-time updates in the PROGRESS.md file
